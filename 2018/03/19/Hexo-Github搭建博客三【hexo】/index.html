Hexo-Github搭建博客三【hexo】

发表于 2018-03-19 | 分类于 搭建博客 | 阅读次数:

Welcome to Hexo! This is your very first post. Check documentation for more info. If you get any problems when using Hexo, you can find the answer in troubleshooting or you can ask me on GitHub.

Before Start

dependent

Installing Hexo is quite easy. However, you do need to have a couple of other things installed first:

· Node.js
· Git

If your computer already has these, congratulations! Just install Hexo with npm:

Install hexo 安装Hexo

Setup

Once Hexo is installed, run the following commands to initialise Hexo in the target .

Once initialised, here’s what your project folder will look like:
新建完成后，指定文件夹的目录如下：

_config.yml

Site configuration file. You can configure most settings here.
网站的配置信息，你可以在此配置大部分的参数。注意这是整个网站的配置信息。

package.json

Application data. The EJS, Stylus and Markdown renderers are installed by default. If you want, you can uninstall them later.
应用程序的信息。因为Hexo工程相当于Nodejs下的一个应用程序，package.json里记录了这个应用程序的信息，包括依赖的其他库的信息。

scaffolds

Scaffold folder. When you create a new post, Hexo bases the new file on the scaffold.
脚本 文件夹。脚本是扩展 Hexo 最简易的方式，在此文件夹内的 JavaScript 文件会被自动执行。

source

Source folder. This is where you put your site’s content. Hexo ignores hidden files and files or folders whose names are prefixed with _ (underscore) - except the _posts folder. Renderable files (e.g. Markdown, HTML) will be processed and put into the public folder, while other files will simply be copied.
资源文件夹。这是存放你的资源的地方。除 _posts 文件夹之外，开头命名为 _ (下划线)的文件 / 文件夹和隐藏的文件将会被忽略。Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。

themes

Theme folder. Hexo generates a static website by combining the site contents with the theme.
主题 文件夹。Hexo 会根据主题来生成静态页面。

Commands 命令

init hexo webside 初始化站

初始化一个站。如果没有设置 folder ，Hexo 默认在当前的目录下建立网站。

Create a new post 创建新文章

More info: Writing

Run server 运行

Hexo 3.0 把服务器独立成了个别模块，您必须先安装 hexo-server 才能使用。

Run server

More info: Server

Generate static files 构建静态页面

More info: Generating

Deploy to remote sites 部署到远程库

More info: Deployment

Debug mode 调试模式

clean 清理缓存db.json 和 构建的public下的文件

Cleans the cache file (db.json) and generated files (public).

Front-matter

Settings & Their Default Values

参数 描述 默认值
layout 布局 -
title 标题 -
date 建立日期 文件建立日期
updated 更新日期 文件更新日期
comments 开启文章的评论功能 true
tags 标签（不适用于分页） -
categories 分类（不适用于分页） -
permalink 覆盖文章网址 -

Categories & Tags

Only posts support the use of categories and tags. Categories apply to posts in order, resulting in a hierarchy of classifications and sub-classifications. Tags are all defined on the same hierarchical level so the order in which they appear is not important.
只有文章支持分类和标签，分类具有层级概念，标签没有层级概念，

If you want to apply multiple category hierarchies, use a list of names instead of a single name. If Hexo sees any categories defined this way on a post, it will treat each category for that post as its own independent hierarchy.
